# PDFConvertOCR Parking Lot

Deferred feature ideas for the conversion-only Windows Explorer tool. Do not
expand the product into a PDF viewer or editor.

## Suggested Priority

1. **Safety and reliability — completed 2026-07-17:** collision-proof names,
   verified output, and original-file choices.
2. **OCR quality — completed 2026-07-17:** understandable presets, optional
   rotation, and installed-language selection.
3. **Workflow convenience:** optional sidecar text, per-folder profiles, and
   an explicitly configured watch folder.
4. **Specialized output — completed 2026-07-17:** PDF/A archival mode and a
   small-file mode.

## 1. Safety and Reliability

**Completed 2026-07-17.** The release added non-overwriting numbered outputs,
temporary staging plus strict validation, `move`/`copy`/`keep` original-file
actions, and richer conversion summaries. See the test coverage in
`tests/test_safety_reliability.py`.

| Status | Idea | Reference |
| --- | --- | --- |
| Done | Original-file handling choice — keep in place, move to `Originals`, or copy there after verified conversion. | [virantha/pypdfocr](https://github.com/virantha/pypdfocr) |
| Done | Collision-proof names — never overwrite; use numbered output siblings and unique archive names. | [virantha/pypdfocr](https://github.com/virantha/pypdfocr) |
| Done | Output verification and conversion summary — validate staged output before publication and report verification, paths, actions, and sizes. | [OCRmyPDF](https://github.com/ocrmypdf/OCRmyPDF) |

## 2. OCR Quality

**Completed 2026-07-17.** Explorer now opens a conversion-only prompt for
presets and detected OCR languages; command-line and batch users can select
the same values without a prompt.

| Status | Idea | Reference |
| --- | --- | --- |
| Done | Clear quality presets — Standard, Straighten and rotate, Archival PDF/A, and Small file. | [OCRmyPDF](https://github.com/ocrmypdf/OCRmyPDF), [Unrud/djpdf](https://github.com/Unrud/djpdf) |
| Done | Straighten and rotate — adds orientation correction on top of Standard's deskew behavior. | [OCRmyPDF](https://github.com/ocrmypdf/OCRmyPDF) |
| Done | OCR language selection — English default plus any detected installed packs, including multi-language combinations. | [OCRmyPDF](https://github.com/ocrmypdf/OCRmyPDF) |

Avoid making aggressive image-cleaning the default. It should remain an
advanced opt-in option only after compatibility testing.

## 3. Workflow Convenience

| Idea | Notes | Reference |
| --- | --- | --- |
| Optional sidecar text file | On request, save extracted OCR text as a `.txt` file next to the resulting PDF. | [aaronmendonplasek/OCR_PDFs](https://github.com/aaronplasek/OCR_PDFs) |

## 4. Specialized Output

**Completed 2026-07-17.** PDF/A is generated by OCRmyPDF without later page
numbering; small-file mode lowers JPEG quality without enabling aggressive
cleaning or downsampling.

| Status | Idea | Reference |
| --- | --- | --- |
| Done | PDF/A archival preset — OCRmyPDF-generated archival output, not independently certified by this app. | [OCRmyPDF](https://github.com/ocrmypdf/OCRmyPDF) |
| Done | Small-file preset — optimize level 3 with JPEG quality 25. | [Unrud/djpdf](https://github.com/Unrud/djpdf) |

## Review Rule

Before implementation, validate a candidate idea against real representative
PDFs for OCR quality, output validity, file size, runtime, and original-file
safety. Keep changes within conversion, validation, and file workflow scope.
